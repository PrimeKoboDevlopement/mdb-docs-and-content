工作原理

Bootstrap的网格系统使用一系列容器，行和列来布局和对齐内容。它内置弹性盒并且反应灵敏。下面是一个示例，并深入研究了网格如何组合在一起。

分解一下，它是这样工作的：

  • 容器提供了一种使网站内容居中的方法。采用 mdb-ontainer 用于固定宽度或 mdb容器与流体支持宽度：100％ 所有视口和设备尺寸。
  • 行是列的包装。每列有水平填充 （称为装订线）以控制它们之间的空间。这个填充然后在具有负边距的行上抵消。这样，您列中的所有内容在视觉上都在左侧向下对齐。
  • 在网格布局中，必须将内容放置在列内，并且只有列可以是行的直接子代。
  • 多亏了flexbox，没有指定网格的列宽度将自动按等宽列布局。例如，四个实例 sm =“ 4” 会自动从小断点开始向上增加25％的宽度。看到自动布局列部分以获取更多示例。
  • 列类表示每行可能使用的12列中的列数。因此，如果您需要三个等宽列，则可以使用 .col-sm-4 。
  • 柱宽度 s是按百分比设置的，因此它们相对于其父元素始终是可变的且大小合适。
  • 列有水平填充在各个列之间创建装订线，但是，您可以删除余量从行和填充从列 .no-gutters 在行 。
  • 为了使网格响应，有五个网格断点，每个断点一个响应断点 ：所有断点（超小），小，中，大和特大。
  • 网格断点基于最小宽度的媒体查询，这意味着它们适用于该一个断点及其上方的所有断点 （例如， sm =“ 4” 适用于小型，中型，大型和超大型设备，但不适用于第一个 xs 断点）。
  • 您可以使用预定义的网格类（例如 col =“ 4” ） 要么无礼的混蛋以获得更多的语义标记。

注意局限性和 flexbox周围的错误 ，如无法将某些HTML元素用作Flex容器 。

自定义网格

使用我们的内置网格Sass变量和映射，可以完全自定义预定义的网格类。更改层数，媒体查询维度和容器宽度，然后重新编译。

列和装订线

网格列的数量及其水平填充（即装订线）可以通过Sass变量进行修改。 $ grid-columns 用于生成每列的宽度（以百分比为单位），而 $ grid-gutter-widths 允许特定于断点的宽度被平均分配左填充和向右填充对于列装订线。

$ grid-columns：12！默认; $ grid-gutter-width：30px！默认

网格层

除了列本身之外，您还可以自定义网格层数。如果您只需要三个网格层，则可以更新 $ grid-breakpoints 和 $ container-max-widths 像这样：

$ grid-breakpoints：（xs：0，sm：480px，md：768px，lg：1024px）; $ container-max-widths：（sm：420px，md：720px，lg：960px）;

对Sass变量或映射进行任何更改时，您需要保存更改并重新编译。这样做将为列宽，偏移量和顺序输出一组全新的预定义网格类。响应性可见性实用程序也将更新为使用自定义断点。确保在中设置网格值像素 （不雷姆 ， EM ，要么 ％ ）。

Vue警报-API

在本节中，您将找到有关如何正确使用警报的信息。

基本用法

类似Toastr的Alert功能在一般组件人群中非常特殊，因为它是一种功能（如果可能的话，是一个插件），而不是模板标记。

这意味着要显示警报，您不必导入警报并自行定位。相反，它可以作为插件使用，这意味着所有Vue实例都可以调用警报。让我们从将这种功能添加到他们的武器库开始。要使警报对所有人可用（也就是将其添加为插件），请将其导入到您的根Vue实例中并使其成为Vue 。采用（） AlertPlug。如果您正在使用 Vue-Cli ，根Vue实例可在 /src/main.js 文件。一般情况下： 

从&#39;vue&#39;导入Vue；导入&#39;bootstrap-css-only / css / bootstrap.min.css&#39;;导入&#39;mdbvue / build / css / mdb.css&#39;;从&#39;导入应用。/App.vue&#39;从&#39;mdbvue&#39;导入AlertPlug; Vue.use（AlertPlug）;

瞧！ 现在，警报无处不在！

发出警报

通常，一旦我们调用其中一个 $警报全局属性方法，即 $ alert.success（）; //或this。$ alert.info（）; //或this。$ alert.wargning（）; //或this。$ alert.danger（）; 如果不带任何参数，则调用它会显示指定颜色的基本警报，并且默认位置（顶部居中）中没有任何内容。

定制化

要给通知一些字符，请传入一个配置对象作为function参数。以下是其可用属性：

名称 类型 默认 描述
标题 字符串 '' 标题（如果有）
信息 字符串 '' 警报的“段落”中的文字
closeButton 真假值 如果为 true，则幻灯片将没有固定位置 关闭按钮（“ x”）应该在那里吗？
progressBar 真假值 false 设置进度条是否应标记警报的超时
|||UNTRANSLATED_CONTENT_START|||position|||UNTRANSLATED_CONTENT_END||| 字符串 右上 可用值：“右上方”，“右下方”，“左下方”，“左上方”，“顶部中央”，“底部中央”，“顶部全宽”，“底部全宽”
showMethod 字符串 淡入 动画制作显示的类名称，默认为淡入
hideMethod 字符串 淡出 动画制作隐藏的类名称，默认为消退
showDuration 号码 1000 放映动画的持续时间女士
hideDuration 号码 1000 隐藏动画的持续时间女士
延迟 号码 0（无输入） 帮助设置警报应在何时出现的延迟女士
超时 号码 1500 设置警报进入的持续时间女士
颜色 字符串 successColor 覆盖警报的类型默认颜色

一个完整的例子这。$警报在某个组件的方法中包含一个配置对象：

this。$ alert.warn（{消息：“这是某个组件内部的程序化警告”，位置：“右上方”，超时：5000，closeButton：false，progressBar：true，延迟：2000}）；

关闭警报

超时后，警报消失，其中涵盖了大多数用例。要关闭警报，用户可以单击默认情况下提供的关闭按钮。最后，应用程序的逻辑可以以编程方式清除所有警报，

this。$ alert.removeAll（）
